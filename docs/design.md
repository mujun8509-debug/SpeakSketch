# SpeakSketch 设计文档

## 1. 项目简介

SpeakSketch 是一个纯语音控制的结构化绘图工具，用户通过自然语言指令在画布上绘制图形，无需手动操作。

**核心特点**：
- 语音驱动：完全通过语音指令控制绘图
- 结构化绘制：所有图形均为可编辑的 Fabric.js 对象
- 智能解析：支持简单图形、复杂对象、复杂场景、空间关系
- 历史管理：完整的撤销/重做/重放功能

## 2. 用户故事

| 用户角色 | 需求描述 | 对应功能 |
|----------|----------|----------|
| 普通用户 | 通过语音绘制基础图形 | 基础绘图指令 |
| 创意用户 | 绘制复杂场景和角色 | 复杂对象和场景 |
| 设计师 | 调整和修改绘制内容 | 编辑指令 |
| 演示者 | 展示创作过程 | 重放功能 |

## 3. 指令能力设计

### 3.1 基础图形
- 圆形、矩形、三角形、直线、文本
- 支持颜色和位置指定

### 3.2 复杂对象
- 人物、猫、狗、汽车、花、山、河、船、草地、鸟

### 3.3 复杂场景
- 公园（人、树、花、草地、鸟）
- 海边（太阳、云、船、海平线、飞鸟）
- 校园（教学楼、树、草地、学生）

### 3.4 空间关系
- 旁边、左边、右边、上方、下方
- 天空中、地面上、河上

### 3.5 编辑指令
- 改色、移动、缩放、删除

### 3.6 系统指令
- 撤销、重做、清空、导出、重放

## 5. 核心模块说明

### 5.1 localParser.ts
- 中文自然语言指令解析器
- 将中文指令转换为结构化的 DrawCommand actions
- 支持颜色、位置、数量等参数提取

### 5.2 relationParser.ts
- 空间关系模板解析器
- 识别"旁边"、"左边"、"天空中"等空间关系表达
- 输出包含位置信息的多个 actions

### 5.3 shapeFactory.ts
- 复杂对象组合绘制工厂
- 使用 fabric.Group 实现简笔画风格组合图形
- 支持人物、猫、狗、汽车等对象

### 5.4 commandExecutor.ts
- 指令执行器
- 协调解析结果与绘图动作的执行流程
- 处理 add/remove/clear/modify 操作

### 5.5 historyManager.ts
- 历史管理器
- 支持撤销、重做、重放功能
- 记录 add/remove/clear/modify 四种历史类型

### 5.6 objectVocabulary.ts
- 对象词汇映射库
- 维护中文词汇到绘图动作类型的语义映射
- 支持动态扩展对象类型

## 6. 成本控制策略

SpeakSketch 采用本地优先的成本控制策略：

| 策略 | 说明 |
|------|------|
| 本地解析 | 默认指令解析在浏览器端完成，不依赖云端服务 |
| 本地绘制 | 使用 Fabric.js 本地渲染，无需服务器资源 |
| 按需调用 | 云端 ASR 与 Seedream 图生图风格化均为可选后端代理能力，仅在用户主动触发并配置接口后调用 |
| Mock/兜底模式 | 未配置后端接口时，通过 Web Speech API、调试输入框或 Mock 模式保证基础功能可用 |

### 6.1 核心成本控制要点

1. **绘图过程本地完成**：结构化草图由本地 Fabric.js 对象生成
2. **简单指令本地解析**：通过 localParser 处理中文指令
3. **复杂对象预定义**：通过 shapeFactory 生成组合图形
4. **空间关系模板化**：通过 relationParser 模板解析
5. **浏览器语音输入**：使用 Web Speech API 作为默认方案
6. **后端可选接入**：云端 ASR 和 Seedream 图生图风格化通过后端代理接口预留，前端不保存 API Key

## 8. 未完成能力与原因

### 8.1 未完成功能

| 功能 | 原因 |
|------|------|
| 开放式抽象语义理解 | 需要大模型支持，成本较高 |
| 复杂遮挡和姿态推理 | 需要计算机视觉能力，超出当前范围 |
| 自由手绘笔刷 | 需要复杂的交互实现 |
| 图层管理 | 当前架构已满足基本需求，优先级较低 |
| 文生图贴纸模式 | 非结构化绘图，与项目定位不符 |

### 8.2 技术约束

1. **浏览器兼容性**：Web Speech API 在不同浏览器中表现不一致
2. **移动端支持**：语音识别在移动端浏览器支持有限
3. **性能限制**：复杂场景大量对象时可能卡顿
